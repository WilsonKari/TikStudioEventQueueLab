# Documentación del proyecto

[`PROJECT_CONTEXT.md`](../PROJECT_CONTEXT.md) es la referencia autoritativa del estado
operativo actual de `main`.

`docs/history/` conserva snapshots históricos de la evolución del proyecto. Estos
documentos pueden incluir nombres, conteos y estados ya superados, por lo que no deben
usarse como baseline sin reconciliarlos primero con el `HEAD` real de `main`.

[`Tests/README.md`](../Tests/README.md) documenta los runners vigentes, la organización
de las suites y sus conteos certificados.
